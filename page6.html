Overall Conclusion

Throughout this project, our group compared three different programming languages: JavaScript, Python, and Java.

The aim was to evaluate how effectively AI could generate a simple rhythm game in each language. Instead of focusing only on whether the generated code worked, we also looked at factors such as the ease of setting up each language, gameplay smoothness, timing accuracy, user interface quality, and the amount of debugging required after the code was generated.

One of the main objectives of this project was to evaluate the usefulness of AI as a programming assistant. We found that AI was capable of producing working code for all three programming languages within a short amount of time. This demonstrated that AI can significantly reduce development time by generating a strong starting point for programmers. However, none of the generated programs were immediately ready for use without modifications.

Key Findings

Our testing showed that each programming language had its own advantages and disadvantages. JavaScript produced the smoothest gameplay experience and required the least amount of setup because the program could run directly in a web browser. Python was the easiest language to understand and modify, making it suitable for beginners, although it did experience timing inconsistencies and audio synchronization issues during gameplay. Java required the longest setup process because additional software and dependencies needed to be installed before the program could run correctly. However, once configured, Java provided stable performance throughout testing.

Every version generated by AI required some level of debugging, testing, and improvement before it performed as expected. Several issues, including inaccurate timing, gameplay inconsistencies, user interface adjustments, and installation problems, had to be solved manually. This highlights that AI should be viewed as a tool that assists developers rather than replacing them completely. Human knowledge remains essential for producing reliable software.

Recommendations

Based on the results of this project, we have several recommendations for future students or developers who plan to use AI when creating software applications.

  • Always test AI-generated code carefully instead of assuming it is completely correct.
  • Provide detailed prompts so the AI understands the required features and expected behaviour.
  • Select the programming language according to the project's requirements instead of choosing only the language that is most familiar.
  • Allow sufficient time for debugging because AI-generated programs often require improvements before they are suitable for practical use.
  • Work collaboratively when reviewing AI-generated code, as different team members may identify different problems and possible solutions.

Lessons Learned

Throughout this project, our team gained practical experience in planning, coding, testing, debugging, and evaluating software generated with artificial intelligence. We also improved our understanding of the strengths and limitations of different programming languages while learning the importance of teamwork, communication, and problem-solving. These experiences demonstrated that successful software development depends not only on programming knowledge but also on careful planning, continuous testing, and effective collaboration.

Our Team

This project was completed through the combined efforts of every member of the group. Each member was responsible for different tasks, including research, coding, testing, documentation, webpage development, and presentation preparation. By dividing responsibilities according to individual strengths, the group was able to complete the project more efficiently while maintaining consistent communication throughout the development process.

Working together also allowed us to share ideas, solve technical challenges, and review each other's work before finalising the project. Collaboration played an important role in ensuring that the final website and case study accurately reflected our findings. Overall, this project strengthened both our technical abilities and our teamwork skills, which will be valuable in future software development projects.

Final Reflection

In conclusion, this project demonstrated that AI can be an effective tool for assisting software development when used responsibly. Although AI was able to generate functional code quickly, human input remained crucial for testing, debugging, and improving the final product.

By completing this project, our team developed a better understanding of programming languages, artificial intelligence, and collaborative software development, making this a valuable learning experience.
