Breaking down what each language actually did well and the flaws

JavaScript + HTML

  • Ran at 144 FPS, smooth and consistent.
  • Cleanest UI out of all three.
  • Most accurate timing windows.
  • Osu map reader had some issues at first.

Python

  • Ran at 700+ FPS but looked choppy.
  • Very easy to read and write the code.
  • Easiest setup out of all three.
  • Audio and visuals were not syncing properly.
  • Input timing was inconsistent.

Java

  • Once it was running, it was stable.
  • No crashed during gameplay.
  • Setup took a really long time.
  • Had to install JDK, configure IDE, resolve dependecies... etc.
  • UI was less polished compared to the others.
