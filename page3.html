Development Journey

This study investigates whether Artificial Intelligence can generate a functional Vertical Scrolling Rhythm Game (VSRG) using different programming languages. The development process was conducted under the same conditions to ensure fair comparison.

Methodology

  1. Each team member selected a different programming language for AI-generated development:
    • Python
    • Java
    • Javascript + HTML
  2. A detailed prompt was prepared by Mr. Adha to generate a Vertical Scrolling Rhythm Game (VSRG) similar to osu!mania. The generated programs were required to include:
    • Falling notes
    • Keyboard input detection
    • Scoring system
    • Accuracy judgement
    • Timing windows
  3. The generated source code was executed in Visual Studio Code.
  4. Each version was tested using the same gameplay scenario and fixed note patterns.
  5. Observations were recorded:
    • Whether the program runs successfully
    • Number of bugs encountered
    • Gameplay responsiveness
    • User interface quality
    • Performance and smoothness
  6. Results from all implementations were compared using identical testing criteria.
  7. Additional testing was performed to determine whether AI could generate a converter capable of reading osu! beatmap formats.

Objectives of the Experiment

Main Goal

Determine whether AI can successfully generate a playable rhythm game across different programming languages.

Comparison Factors

  • Functionality
  • Code quality
  • Accuracy
  • Performance
  • User experience
